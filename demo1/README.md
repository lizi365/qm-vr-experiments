# Quantum VR — Demo 1（粒子渲染压测，含Demo 0基线）

这一版把 Demo 0 的立方体也保留了（挪到左上角，持续旋转），所以**这一次部署只需要测一次，就能同时拿到基线FPS和粒子压测数据**，不用分两次上头显。

## 场景里有什么

- **左上角小立方体**：Demo 0 的原班底，作为性能基线参照物，全程持续旋转
- **正前方粒子球**：模拟电子云的静态分布，用斐波那契球面算法生成，内蓝外白的渐变色模拟"能量"分布
- **头顶悬浮面板**：实时显示当前粒子数 + FPS，头显里直接能看到，不用摘头显看电脑
- **手柄射线**：从手柄延伸出的淡蓝色细线，帮助你判断手柄追踪是否正常响应

## 操作方式

- **进入VR后**，随便哪只手柄按**扳机键（select / trigger）**，粒子数会按档位跳到下一级：
  `500 → 2,000 → 8,000 → 20,000 → 50,000 → 100,000` → 循环回到500
- **桌面浏览器预览**（不戴头显先看效果）：按数字键 `1`-`6` 直接跳到对应档位

## 部署方式

见仓库根目录 README 的部署步骤说明（GitHub → Cloudflare Pages，纯静态文件，构建命令留空）。Demo 0 和 Demo 1 是同仓库不同路径（`/demo0/` 和 `/demo1/`），两个链接都留着，方便随时回去对比基线。

## 如果100,000档位都没掉帧

说明这台Pico企业版的粒子渲染余量比预期大，我们可以直接跳到更有意思的测试——比如给每个粒子加动态shader效果（呼吸感、颜色随时间变化），而不是继续加纯静态粒子数，因为静态粒子数上限已经不是瓶颈了。

## 如果很早就掉帧（比如8,000档就跌破90fps）

说明这台设备的移动端GPU余量比较紧张，之后的demo设计要更保守地控制同屏渲染对象数量，粒子效果可能要靠"更少粒子+更强shader"的思路来营造视觉密度，而不是堆数量。
